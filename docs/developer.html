Install Usage Developer FAQ

Developer Guide

Prepare a Dockan app so an admin can install Dockan, build the app locally, and run it as a Linux service.

1. Developer

Prepare `Dockanfile`, `dockan.yml`, code, and README.

2. Share

Publish through Git, a `.tar.gz` archive, an internal server, or offline media.

3. Admin

Install Dockan, build locally, run the app, and create the service.

Developer

Prepare The Project

Recommended structure:

myapp/
  Dockanfile
  dockan.yml
  README.md
  app.sh
  src/

Dockanfile

FROM scratch
LABEL org.opencontainers.image.title=MyApp
COPY app.sh /app.sh
RUN chmod +x /app.sh
CMD ./app.sh

dockan.yml

name: myapp
services:
  web:
    build: .
    image: myapp:latest
    ports:
      - 8080:8080
    volumes:
      - data:/data
    restart: always
Admin

Install And Run Locally

Install Dockan on the Linux machine:

curl -fsSL https://raw.githubusercontent.com/Dockan-Conteneurisation-libre/Dockan/main/scripts/install.sh | sh
dockan doctor

Fetch the app, build it, and run it:

tar -xzf myapp-dockan-v1.tar.gz
cd myapp
dockan build -t myapp:v1 .
dockan compose up

Check status and logs:

dockan ps -a
dockan logs myapp-web
Service

Start On Boot

Native system auto-start:

sudo dockan compose autostart -f /srv/myapp/dockan.yml --name myapp

User service:

dockan compose autostart --user -f ~/myapp/dockan.yml --name myapp

Disable auto-start:

sudo dockan compose no-autostart -f /srv/myapp/dockan.yml --name myapp
Local Bases

Share A Full Base

If the app needs an Ubuntu, Alpine, or Python base, the developer can provide a rootfs archive.

dockan base import ubuntu:local ./ubuntu-rootfs.tar.gz
dockan build -t myapp:v1 .

The `Dockanfile` can then use the local base:

FROM ubuntu:local
COPY app.sh /app.sh
CMD ./app.sh
Network

Ports And Network

Simple network:

dockan network create appnet
dockan run -d --name web --network appnet myapp:v1

Bridge/NAT with port publishing:

dockan network create appnet --driver bridge --subnet 10.89.0.0/24 --gateway 10.89.0.1/24 --bridge dockan0
sudo dockan network enable appnet
sudo dockan run -d --name web --network appnet -p 8080:80 myapp:v1
Summary

What To Remember

  • The developer shares the app: `Dockanfile`, `dockan.yml`, code, and README.
  • The admin installs Dockan, builds locally, and runs locally.
  • For automatic startup, Dockan creates and enables a native systemd service.
  • GitHub Actions release packages build Dockan itself, not user apps.
