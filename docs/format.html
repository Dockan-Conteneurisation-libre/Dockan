Dockan Format

Dockanfile, simple Dockerfile support, local bases, and the legacy `.dockan` folder format.

Recommended Format: Dockanfile

FROM scratch
LABEL org.opencontainers.image.title=MyApp
WORKDIR /app
COPY . /app
RUN chmod +x ./start.sh
CMD ./start.sh
dockan build -t myapp:latest .
dockan run myapp:latest

Supported Instructions

  • FROM, COPY, ADD, RUN, CMD, ENTRYPOINT
  • ENV, WORKDIR, EXPOSE, VOLUME, LABEL, ARG
  • USER, SHELL, STOPSIGNAL, HEALTHCHECK

Dockan also supports `.dockerignore`, simple multi-stage builds, `COPY --from=...`, and Docker exec-form commands such as CMD ["node", "server.js"].

Local Bases

dockan base import alpine:local ./alpine-rootfs
dockan base import ubuntu:local ./ubuntu-rootfs.tar.gz
FROM alpine:local
COPY app.sh /app.sh
CMD ./app.sh

Legacy .dockan Folder

myapp.dockan/
  meta.conf
  build.sh
  start.sh
  rootfs/
  hooks/
  volumes/
  logs/

For new projects, prefer `Dockanfile`.
