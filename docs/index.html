Dockan

A local Docker alternative for Linux: daemonless, easy to install, built for running apps, keeping data local, and understanding what runs on the machine.

Install GitHub 
curl -fsSL https://raw.githubusercontent.com/Dockan-Conteneurisation-libre/Dockan/main/scripts/install.sh | sh

dockan doctor
dockan build -t hello:latest examples/hello
dockan run hello:latest

Install

Dockan installs from GitHub Releases. The installer downloads the right Linux package and verifies checksums when they are available.

User

Install without sudo

curl -fsSL https://raw.githubusercontent.com/Dockan-Conteneurisation-libre/Dockan/main/scripts/install.sh | sh

If Dockan installs into ~/.local/bin, add it to your shell path and verify the CLI:

export PATH="$HOME/.local/bin:$PATH"
dockan version
dockan doctor
dockan ps -a
System

System-wide install

curl -fsSL https://raw.githubusercontent.com/Dockan-Conteneurisation-libre/Dockan/main/scripts/install.sh | sudo sh
dockan update
dockan update --version v0.1.1
dockan update --system

Express update from a local source checkout:

cd /home/anar/Bureau/Test/Dockan && git pull && make install PREFIX=/home/anar/.local

Dependencies

Dockan can install the host packages it needs through apt, dnf, apk, pacman, or zypper. It only installs packages when you explicitly run a deps command.

Core

Normal operation

Use core for the base tools Dockan needs: shell, curl, certificates, archives, process tools, and network helpers.

dockan deps install --dry-run core
sudo env "PATH=$HOME/.local/bin:$PATH" dockan deps install core -y
Full

Recommended host setup

Use full to install the recommended modern host setup: core tools, utilities, Node/npm frontend tools, network helpers, database clients, Nginx, Caddy, build tools, diagnostics, and isolation helpers.

dockan deps install --dry-run full
sudo env "PATH=$HOME/.local/bin:$PATH" dockan deps install full -y
Runtime

FrankenPHP for PHP apps

Dockan can prepare FrankenPHP automatically. On Fedora it configures the static PHP repository and installs frankenphp through dnf.

sudo env "PATH=$HOME/.local/bin:$PATH" dockan deps runtime frankenphp -y
command -v frankenphp
Profiles

Package groups

  • core: required base packages
  • tools: git, jq, archives, rsync, OpenSSL, GPG, lsof, and file tools
  • frontend: Node.js and npm for React, Vue, Vite, and other Node frontends
  • network: network and process helpers
  • database: common database client tools
  • web: Nginx and Caddy for local reverse proxy workflows
  • build: compiler and package-config tools
  • debug: strace, psmisc, htop, and tcpdump
  • isolation: sandbox helpers when available
  • full: recommended complete setup
dockan deps install --dry-run frontend
dockan deps install --dry-run database
sudo env "PATH=$HOME/.local/bin:$PATH" dockan deps install frontend database web build debug -y

Profiles install the distribution's default package versions. To request a specific package version, pass the native package-manager syntax directly.

sudo dockan deps install --manager apt -y 'nodejs=20.*'
sudo dockan deps install --manager dnf -y nodejs-20.11.1
sudo dockan deps install --manager apk -y 'nodejs=20.11.1-r0'
sudo dockan deps install --manager zypper -y nodejs-20.11.1

For strict runtime versions, import a prepared local base such as dockan base runtime node:20 --from ./node20-rootfs.tar.gz. If sudo dockan says command not found after a user install, keep the sudo env "PATH=$HOME/.local/bin:$PATH" form or use the absolute Dockan path.

Dockan Panel

Dockan Panel is the optional browser UI for containers, images, volumes, stacks, backups, and live terminals.

Open Dockan Panel repository

Storage

Persistent panel data

The panel stores users, password hashes, 2FA/TOTP secrets, passkey public keys, stacks, and panel backups in the dockan-panel-data volume.

/app/storage/auth-users.json
~/.local/share/dockan/volumes/dockan-panel-data

Removing that volume removes the panel users, 2FA, passkeys, stacks, and panel backups.

Run

Start the UI

Dockan Panel runs with the host frankenphp binary. Its Dockan image is scratch, and the panel command is frankenphp run --config Caddyfile.

command -v frankenphp || sudo env "PATH=$HOME/.local/bin:$PATH" dockan deps runtime frankenphp -y
cd /path/to/Dockan-Panel
dockan compose up

Open http://127.0.0.1:9090, then create the first admin account. There is no default password and no default token.

For public access, put an HTTPS reverse proxy in front of the panel. Do not expose port 9090 directly to the Internet over plain HTTP.

Passkeys work on localhost, 127.0.0.1, or HTTPS. Browsers usually block passkeys on plain HTTP LAN addresses.

Core Usage

A `Dockanfile` is enough to build a local image and run it.

Build

Create an image

FROM scratch
LABEL org.opencontainers.image.title=MonApp
COPY app.sh /app.sh
RUN chmod +x /app.sh
CMD ./app.sh
Run

Run and manage

dockan build -t monapp:latest .
dockan run -d --name monapp monapp:latest
dockan logs monapp
dockan exec monapp sh
dockan stop monapp

Dockan vs Docker

Dockan is a real Docker alternative for local Linux workflows. It focuses on simple commands, local files, no permanent daemon, and no required cloud registry.

Dockan

Why choose Dockan

  • Daemonless local app runner
  • Folder-based local registry
  • Simple Dockanfile and Dockerfile-style builds
  • Systemd service install for servers
  • Readable files, archives, and checksums
  • Good fit for self-hosting, labs, education, and internal tools
Docker

Where Docker is stronger

  • Full Docker Hub ecosystem
  • Mature OCI layers and cache behavior
  • Full Dockerfile compatibility
  • Dynamic internal DNS and mature networking
  • Large production fleets already standardized on Docker
  • Largest third-party tooling ecosystem

Short version: Dockan is simpler and more local. Docker is broader and more mature.

Available Features

Images

Dockanfile and simple Dockerfile

Common instructions, `.dockerignore`, local bases, and simple multi-stage builds with `COPY --from`.

Registry

Local push and pull

Use a folder as a local registry with image archives, checksums, and `dockan push` / `dockan pull`.

Languages

Any Linux runtime

PHP, Node.js, Python, Go, Rust, Java, Ruby, shell scripts, and static binaries through host runtimes, local bases, or shipped binaries.

Containers

Run, logs and exec

Foreground or detached runs, inspection, stop, remove, and commands inside a running container.

Volumes

Persistent data

Named volumes, local folder mounts, and tar.gz backup/restore for app data.

Compose

Multiple services

`dockan compose up` reads a `dockan.yml` with build, image, ports, env, volumes, aliases, depends_on, restart, healthcheck, memory, and cpus.

Health

Healthchecks

Use `dockan health` or `dockan compose health` to run service readiness checks.

Service

systemd

Install as a system or user service so an app can start on boot.

Local

No forced cloud

Dockan does not automatically download from Docker Hub. Bases and files stay on your machine.

Known App Example

A WordPress-style PHP app with a MariaDB/MySQL database shows the practical Dockan workflow: local images, compose, network aliases, volumes, healthchecks, and backup.

WordPress

Local app + DB

name: wordpress
services:
  web:
    image: wordpress:local
    ports:
      - 8080:8080
    env:
      - DB_HOST=db
      - DB_NAME=wordpress
      - DB_USER=wordpress
      - DB_PASSWORD=change-me
    volumes:
      - wp-data:/var/www/html
    network: wpnet
    aliases:
      - web
    depends_on:
      - db
    restart: always
    healthcheck: CMD-SHELL curl -f http://127.0.0.1:8080/
  db:
    image: mariadb:local
    env:
      - DB_NAME=wordpress
      - DB_USER=wordpress
      - DB_PASSWORD=change-me
      - DB_ROOT_PASSWORD=change-root
    volumes:
      - db-data:/var/lib/mysql
    network: wpnet
    aliases:
      - db
    restart: always
    healthcheck: CMD-SHELL test -d /var/lib/mysql
networks:
  - wpnet
Run

Operate locally

Use prepared local images such as wordpress:local and mariadb:local. Dockan does not pull them from Docker Hub automatically.

dockan compose up
dockan compose health
dockan volume backup db-data wordpress-db-backup.tar.gz

Network

Dockan can create simple networks. Bridge/NAT with veth and published ports requires `sudo`.

Simple

Local network

dockan network create appnet
dockan run -d --name web --network appnet --alias web.local web:latest
dockan network refresh appnet
dockan network hosts appnet
dockan network doctor
Bridge

Bridge/NAT with ports

dockan network create appnet --driver bridge --subnet 10.89.0.0/24 --gateway 10.89.0.1/24 --bridge dockan0
sudo dockan network enable appnet
sudo dockan run -d --name web --network appnet -p 8080:80 web:latest

Production

Dockan is usable as a simple local container tool. For serious production use, publish releases with checksums, back up volumes, run healthchecks, and test root-level features on a clean Linux machine.

Release

Publish a version

git tag v0.1.0
git push origin v0.1.0
Operate

Health and backup

dockan compose health -f /srv/myapp/dockan.yml
dockan volume backup data /srv/backups/data.tar.gz
Install

Copy-ready command

curl -fsSL https://raw.githubusercontent.com/Dockan-Conteneurisation-libre/Dockan/main/scripts/install.sh | sh

Local Registry

Push and pull images through a local folder. No Docker Hub, no daemon, no cloud service.

dockan push myapp:latest /srv/dockan-registry
dockan registry ls /srv/dockan-registry
dockan pull myapp:latest /srv/dockan-registry
